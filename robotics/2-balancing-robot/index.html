Self-Balancing Robot

Two-wheel self-balancing platform with IMU sensor fusion, PID control, and ROS2 nodes for mapping and person-following.

See Gallery

Overview

Built as a team project: I led system integration and implemented key modules (control loop, ROS2 messaging, perception pipeline).

Project Goals
  • Stable balancing via IMU (gyro+accel) with complementary/kalman fusion.
  • Tunable PID for tilt → motor PWM with safe saturation and anti-windup.
  • ROS2 nodes: sensor stream, command, SLAM (LiDAR + Cartographer).
  • Person detection + follow mode (vision or LiDAR tracking).
Architecture

MCU (ESP32/STM32) handles real-time control → Motor drivers (H-bridge). Companion computer (Raspberry Pi) runs ROS2 (Jazzy), SLAM, and higher-level behaviors. Telemetry via MQTT / ROS2 topics.

Build Notes

Control Tuning
  • Step-response method to seed PID; refined with Ziegler–Nichols.
  • Added feed-forward term for faster recovery from pushes.
  • Complementary filter (98/2 split) before trying EKF.
ROS2 Integration
  • Nodes: /imu, /wheel_odom, /cmd_vel, /map.
  • Cartographer config tuned for short-range LiDAR.
  • MQTT bridge for lightweight telemetry to laptop dashboard.
